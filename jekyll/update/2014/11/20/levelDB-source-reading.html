levelDB源码阅读

Nov 20, 2014

levelDB是基础的Key-Value存储系统，它的存储容器包括内存和磁盘，通过将存储内容按时间分层的原理，在写入的过程中将数据优先存入内存，当内存中的数据到达临界点的时候，将内存的数据持久化到磁盘；在读数据的时候，也是优先从内存查找，在查找失败时向较老的数据中查找。由于写数据是直接操作内存，所以写操作比读操作有更高的效率。出于对这个系统实现的好奇，如何实现高性能读写，以及如何容灾，和事务处理，所以开始阅读它的源码。

mutexlock帮助简化加锁操作

#define EXCLUSIVE_LOCK_FUNCTION(...)
#define UNLOCK_FUNCTION(...)

class MutexLock {
  public:
	explicit MutexLock(port::Mutex *mu) EXCLUSIVE_LOCK_FUNCTION(mu)
		: mu_(mu) {
		this->mu_->Lock();
	}
	~MutexLock() UNLOCK_FUNCTION() { this->mu_->Unlock(); }
  private:
  	port::Mutex *const mu_;
  	// No copying allowed
  	MutexLock(const MutexLock&);
  	void operator=(const MutexLock&);
};

使用方式是申明一个MutexLock对象，并且不保存该指针到变量中，如:

void func() {
	MutexLock l(&mu_); // 申明了一个MutexLock的变量，执行构造函数，即Lock
	some operations...
	//当离开函数时l自动被回收，执行析构函数，即Unlock
}

使用leveldb方法

leveldb实际上是c++写的一个集成库，集成到项目中作为键值系统的一个本地持久化方案，它解决的就是如何高效、并发、安全地将数据持久化到本地存储。
简易的使用示例可以参考leveldb使用初步。

#include <iostream>
#include "leveldb/db.h"
 
using namespace std;
 
int main()
{
    leveldb::DB* db;
    leveldb::Options options;
    leveldb::Status s;
 
    options.create_if_missing = true;
    //options.error_if_exists = true;
    std::string dbpath = "testdb";
    s = leveldb::DB::Open(options, dbpath, &db);
    if (!s.ok()) {
        cerr << s.ToString() << endl;
        return -1;
    }
 
    std::string value;
    s = db->Put(leveldb::WriteOptions(), "k1", "v1");
    cout << s.ok() << endl;
    s = db->Get(leveldb::ReadOptions(), "k1", &value);
    cout << s.ok() << " " << value << std::endl;
    s = db->Delete(leveldb::WriteOptions(), "k1");
    cout << s.ok() << endl;
    value.clear();
    s = db->Get(leveldb::ReadOptions(), "k1", &value);
    cout << s.ok() << "" << value << std::endl;
 
    delete db;
    return 0;
}
